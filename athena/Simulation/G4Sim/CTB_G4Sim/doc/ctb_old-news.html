  • 30/07/04 - Using Atlas 8.5.0 (Geant4-6.1), problems with the G4-navigator were found in different points of the CTB setup (LAr and Muons). Since that we are running with G4navigator check-mode on. This makes that most of the jobs will crash using 8.5.0. The nightlies after 8.5.0 are using Geant4.6.2 and the situation is much better (you should use CTB_G4Sim-00-01-14). Please, try to be sure that the G4-geometry of your subdetector is clean in terms of the G4-geometry recursive test.
  • 14/07/04 - A bug in Geant4-6.2 was found in the multiple scattering process, do not use 8.4.0 Atlas release.
  • 09/07/04 - CTB_G4Sim-00-01-11 works with Atlas 8.4.0 as it is out of the box.
  • 18/06/04 - CTB_G4Sim-00-01-08 works with Atlas 8.3.0 as it is out of the box, with the python jobOption files.
  • 07/06/04 - Starting with the "CTB_G4Sim-00-01-06" tag, CTB_G4Sim will use again the main Atlas line 8.X.0. "CTB_G4Sim-00-01-06" is the last tag that can be run in 8.X.0 and 8.0.X. The next available tag "CTB_G4Sim-00-01-07" will run only in the 8.X.0 main Atlas release (nightly buildings at ATLASSW_OPT).
  • 16/04/04 - Note that for the Atlas Combined test beam 2004 simulation we are using (since 15/04/04) the Atlas branch release 8.0.X and the associated nightly buildings are in NICOS under the project ATLASSW_PROD, the path at CERN is "/afs/cern.ch/atlas/software/dist/nightlies/nightlies/rd/".
